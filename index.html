接⼝口与抽象类区别

1、一个类声明可否既是abstract的,⼜又是final的?
不能,这两个修式符⽭矛盾（abstract就是要被继承）
2、抽象类不不⼀一定包含抽象⽅方法
3、有抽象⽅方法,则⼀一定是抽象类
4、抽象类不不能被实例例化，⼀一般⽤用作基类使⽤用；

a. 类可以实现多个接⼝口但只能继承⼀一个抽象类
b. 接⼝口⾥里里⾯面所有的⽅方法都是Public的，抽象类允许Private、Protected⽅方法
c. JDK8前接⼝口⾥里里⾯面所有的⽅方法都是抽象的且不不允许有静态⽅方法，抽象类可以有普通、静态⽅方法，JDK8 接⼝口可以实现默认⽅方法 和静态⽅方法，前⾯面加default、static关键字。

发表于 2020-09-13 分类于 JavaBasic

继承

  • 继承描述了Java类与类之间的关系，在Java中一个类只能继承另一个类，不过可以实现多级继承
  • 格式： public class 子类名 extends 父类名{ }
  • 下面我们来看两个问题：

继承中，子类继承了父类的哪些属性和方法？
这个答案是： 所有
为什么是所有呢？ 因为这里是一个权限问题，我们只能是说不够权限访问不能说是没有继承，你可以拥有，不一定够权限使用。

Java_使用面向对象设计程序

发表于 2020-09-04 分类于 JavaBasic

类与对象

面向对象
程序：解决问题
网络交易：
    交易：
        卖家（商品）
        买家    （货币）
    网上交易：
        卖家（商品  电话号码  姓名  收款账户  地址 ）
            功能： 开店  上架 标价 处理订单 发货 ...
        买家    （货币 电话号码  姓名  付款账户  收货地址）
            功能： 浏览 加购物车  下单 支付  评价....
      根据用户的需求来创建功能     
  流程：注册-> 注册信息表 —> 账号-> 使用功能
        构造对象    ->    对象名  -> 对象调用方法

概念：
类： 具有相同的、相似的属性、特征、行为方式、功能的一类事物的集合
程序中： 具有相同的、相似的属性、方法的一类对象的代码模板
对象：具体的事物,一个对象就是一个具体的事物
程序中： 由类创建的，存在于内存中

类的编写：
    格式： public  class 类名{ }
  代码命名规范:
      类名：首字母大写，其后的每个单词首字母大写 -- 驼峰命名法
      变量名：属性名   对象名  参数名  变量   
              首字母小写，其后的单词首字母大写，如果缩写可以表达意思就可以缩写
      常量名：全字段大写 EXTI_ON_CLOSE,尽可能的表达这个常量的作用
      方法名：首字母小写，其后的单词首字母大写
      包名： 全小写，每个段之间最好只有一个单词

    一个类需要写什么：

        属性：可以由一个数据，变量表达的
            格式：属性类型  属性变量名；
          类型：
              基本数据类型 ：bit byte  short  int long double  char
              类-数据类型  ： String  只要是一个类就可以作为一种类型
        方法：行为，功能的描述
            格式： 访问修饰符 返回值类型 方法名（参数类型）{ // 方法体}

继承：

    关键字：extends
    格式：public class 子类名 extends 父类名{ }
        父类： 超类 superClass

  Q:子类继承了父类的哪些东西？
  A:      全部             

  Q：子类以及子类对象可以调用父类的哪些属性和方法？
  A:    根据父类和子类的位置，父类属性和方法的访问修饰符来决定子类以及子类对象有没有访问权限
        同包下：
            私有的属性和方法不能被调用
        不同包下：
            不同包的子类下:   私有的和默认的 属性在子类中不能被调用             
            不同包的非子类下:   私有的和默认的以及受保护的 属性不能被子类对象调用

  方法重写：
      在子类中重写父类的方法：
      格式：除方法体之外要写的一致

      子类对象优先调用重写之后的
      不改变父类的代码，的基础上再扩展代码
          可以使用surper来指代父类先调用一次父类的代码

      作用：扩展 丰富

  自动转型：
        类 实现 接口   类的对象可以自动转型为接口类型

     强制转型：
            int i=1.2;  int i=(int) 1.2;
  自动转型：    
        类   对象 = new 类名（）；
        父类 对象 = new 子类名（）；
        子类对象可以自动转型为父类类型
        子类对象转型后不能再调用自己独有的属性和方法了
